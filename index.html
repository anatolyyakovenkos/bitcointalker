Bitcoin: 1 Million Blocks
The Story



A Peer-to-Peer Electronic Cash System

I've been working on a new electronic cash system that's fully peer-to-peer, with no trusted third party.

The network timestamps transactions by hashing them into an ongoing chain of hash-based proof-of-work, forming a record that cannot be changed without redoing the proof-of-work. The longest chain not only serves as proof of the sequence of events witnessed, but proof that it came from the largest pool of CPU power.

Each block contains a collection of new transactions. A block is accepted by the network only if it contains a valid proof-of-work. The chain of blocks is the shared public history that everyone in the network agrees on.

The Block Structure:
Code:
Block Header:
- Version
- Previous Block Hash
- Merkle Root
- Timestamp
- Difficulty Target (Bits)
- Nonce

Block Body:
- Transaction Count
- Coinbase Transaction (Block Reward)
- User Transactions


Block Rewards & Halvings:
The block reward starts at 50 BTC and halves every 210,000 blocks:

- Blocks 0 - 209,999: 50 BTC per block
- Blocks 210,000 - 419,999: 25 BTC per block
- Blocks 420,000 - 629,999: 12.5 BTC per block
- Blocks 630,000 - 839,999: 6.25 BTC per block
- Blocks 840,000 - 1,049,999: 3.125 BTC per block

By block 1,000,000, approximately 19,687,500 BTC will have been mined out of the 21 million total supply. The remaining coins will be mined over the next century through ever-smaller block rewards.

The Journey to 1 Million Blocks:
With an average block time of 10 minutes, 1 million blocks represents approximately 19 years of continuous operation. The difficulty adjustment mechanism ensures that blocks continue to be found roughly every 10 minutes regardless of how much hash power joins or leaves the network.

Every single block is a testament to the security and resilience of this system. Each one builds upon the last, creating an unbreakable chain of mathematical proof. No government, no corporation, no individual can alter the history written into these blocks.

Quote from: nodemaster on November 06, 2008, 08:17:45 AM
How many transactions can fit in a single block? And what happens when we reach 1 million blocks - will the blockchain be too large for regular computers to store?

The current block size allows for a reasonable number of transactions. As technology improves, storage and bandwidth will become cheaper, making it feasible for nodes to store the full chain even at 1 million blocks and beyond.

Moore's Law should ensure that hardware keeps pace with the growth of the blockchain. And even if the full chain becomes large, SPV (Simplified Payment Verification) clients can verify transactions without downloading every block. Only miners and full nodes need the complete history.

The system is designed to scale with time.
